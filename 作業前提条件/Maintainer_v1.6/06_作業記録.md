# 作業記録 - Maintainer用

## 📌 このモジュールについて

**用途**: 作業ログ・レポート作成の詳細、記録管理の方法
**必読条件**: 作業記録を作成する際、レポートを作成する際
**関連モジュール**: 00_QUICK_REFERENCE.md（最重要ルール）、01_基本ルールと役割.md（基本原則）

---

> [!danger] 重要
> Maintainerは以下の作業実施後、**必ず作業記録を作成すること**:
> - アーカイブ実施後
> - ドキュメント大幅更新後
> - 改善提案実施後
> - Workspace全体調査後
> - 問題発見・対処後
> - **Workerへの作業指示書作成後**
>
> 作業完了時に **00_QUICK_REFERENCE.md の「作業完了後チェックリスト」** で必ず確認してください。

---

## 作業記録の重要性

### 最重要ルール

**Maintainerは実施した全ての作業について、詳細な記録を残すこと。**

### 作業記録が必要な理由

1. **トレーサビリティの確保**
   - 過去の作業内容・判断理由を追跡可能にする
   - 「いつ・誰が・何を・なぜ」を明確化

2. **改善効果の測定**
   - 整理・改善の効果を定量的に評価できる
   - Before/Afterの比較が可能

3. **知見の共有**
   - 他のMaintainer（または将来の自分）への引き継ぎ
   - ベストプラクティスの蓄積

4. **問題発生時の原因調査**
   - 「いつ・何を・なぜ変更したか」を明確化
   - 迅速な原因特定と対処

## 作業記録作成のタイミング

### 必須（Must）

以下の作業を実施した場合は、必ず作業記録を作成する：

- ✓ アーカイブ実施後
- ✓ ドキュメント大幅更新後
- ✓ 改善提案実施後
- ✓ Workspace全体調査後
- ✓ 問題発見・対処後

### 推奨（Should）

以下のタイミングでは、作業記録を作成することを推奨：

- ✓ 毎日の作業終了時（作業ログ）
- ✓ 週次・月次レポート作成時
- ✓ 重要な判断を行った時
- ✓ 複数のファイル操作を実施した時

## 定期レポートの作成

### 週次レポート

#### 内容

- 今週作成された日付フォルダ数
- 今週更新されたドキュメント数
- 今週検出された問題数
- 今週実施した整理・改善内容

#### ファイル名と保存先

- **ファイル名**: `YYYYMMDD_週次レポート.md`
- **保存先**: `workspace_management/periodic_reports/weekly/`

---

### 月次レポート

#### 内容

- 月間のファイル数推移
- アーカイブされたフォルダ数
- ドキュメントの更新状況
- 命名規則違反の推移
- 改善提案の実施状況

#### ファイル名と保存先

- **ファイル名**: `YYYYMM_月次レポート.md`
- **保存先**: `workspace_management/periodic_reports/monthly/`

---

### 四半期レポート

#### 内容

- プロジェクト全体の健全性評価
- 主要な改善内容
- 次四半期の改善計画
- 技術選定の見直し提案

#### ファイル名と保存先

- **ファイル名**: `YYYYQX_四半期レポート.md`
- **保存先**: `workspace_management/periodic_reports/quarterly/`

## 記録作成のベストプラクティス

### 1. 具体的に記述する

**Good**:
```
20個の日付フォルダ（20251001〜20251020）をarchive/2025/10-October/に移動。
各フォルダの作業ログを確認し、全て完了ステータスであることを確認済み。
```

**Bad**:
```
古いフォルダをアーカイブしました。
```

### 2. 数値で示す

**Good**:
```
作業前: 150個のファイル
作業後: 130個のファイル
削減: 20個（13%削減）
```

**Bad**:
```
ファイルを整理して少なくなりました。
```

### 3. 根拠を明示する

**Good**:
```
Globツールで「202510*」を検索し、20個のフォルダを特定。
各フォルダの最終更新日時を確認し、全て30日以上経過していることを確認。
```

**Bad**:
```
古いフォルダだったので削除しました。
```

### 4. 次のアクションを明確に

**Good**:
```
次回アクション:
1. 11月分のフォルダも同様にアーカイブ対象か確認（実施予定: 12/1）
2. アーカイブインデックスの更新（実施予定: 11/25）
```

**Bad**:
```
次回も同じようにやります。
```

## 記録の管理

### フォルダ構造

```
workspace_management/
├── reports/                        # 実施レポート・分析レポート
│   ├── YYYYMMDD_Maintainer実施レポート_XX.md
│   └── YYYYMMDD_Maintainer分析レポート_XX.md
├── records/                        # 日々の作業ログ
│   └── YYYYMMDD_Maintainer作業記録_XX.md
└── periodic_reports/               # 定期レポート
    ├── weekly/                     # 週次レポート
    │   └── YYYYMMDD_週次レポート.md
    ├── monthly/                    # 月次レポート
    │   └── YYYYMM_月次レポート.md
    └── quarterly/                  # 四半期レポート
        └── YYYYQX_四半期レポート.md
```

### 古い記録のアーカイブ

#### 基準

- 作業ログ: 3ヶ月経過後
- 実施レポート・分析レポート: 6ヶ月経過後
- 定期レポート: 保持（アーカイブしない）

#### アーカイブ先

```
archive/workspace_management/
├── YYYY/
│   └── reports/
│       └── YYYYMMDD_XX.md
```

## 記録のレビュー

### セルフレビューのポイント

作業記録作成後、以下をチェック：

- [ ] 作業目的が明確か
- [ ] 具体的な数値が含まれているか
- [ ] 根拠が明示されているか
- [ ] 次のアクションが明確か
- [ ] ファイル名が命名規則に従っているか
- [ ] 適切なフォルダに保存されているか

### 定期的な見直し

- **週次**: 今週の作業ログをレビュー
- **月次**: 今月の全記録をレビュー
- **四半期**: 記録の傾向分析、改善点の抽出
