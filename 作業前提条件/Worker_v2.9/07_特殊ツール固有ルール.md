# 特殊ツール固有ルール - Worker用

## 📌 このモジュールについて

**用途**: Adobe After Effects、Unreal Engine等の特殊ツール・プロジェクト固有のルール
**必読条件**: After EffectsやUnreal Engineを扱う作業を行う際に必読
**関連モジュール**:
- `02_ファイル管理.md` - 通常のファイル管理規則
- `03_バッチファイル・スクリプト規約.md` - スクリプト作成規約

---

## 特殊ツール・プロジェクト固有のルール

### 概要
After Effects、Unreal Engine などの特殊なツールやプロジェクトを扱う際の注意事項。
新しいツールを利用する際は、そのツール固有のルールを確認し、このセクションに追記する。

### 現在対象のツール
- Adobe After Effects
- Unreal Engine

---

## Adobe After Effects

### プロジェクトファイルの管理
- `.aep` ファイル: After Effects プロジェクトファイル
- プロジェクトファイルは日付フォルダに格納
- バージョン管理: `project_v1.aep`, `project_v2.aep`

### 素材・アセットの管理
```
20251113\
├── 20251113_worklog.md
├── project_v1.aep
├── assets\                 # 素材フォルダ
│   ├── images\
│   ├── videos\
│   └── audio\
└── log\
    └── render_output\      # レンダリング出力
```

### レンダリング出力
- 出力先: `log/render_output/`
- 命名: `render_20251113_143052.mp4`
- 大容量ファイルは Git にコミットしない

### 注意事項
- プロジェクトの収集機能を使用して素材を一箇所に集約
- レンダリング設定をメモに残す
- 使用したエフェクト・プラグインをドキュメント化

## Unreal Engine

### プロジェクトファイルの管理
- `.uproject`: Unreal Engine プロジェクトファイル
- プロジェクトは通常、専用フォルダに配置
- 日付フォルダには作業ログとスクリプトのみ格納

### フォルダ構造
```
C:\Users\CLPC-63\Documents\Unreal Projects\
└── MyProject\              # Unreal Engine プロジェクト本体
    ├── MyProject.uproject
    ├── Content\
    ├── Source\
    └── Config\

C:\Users\CLPC-63\workspace\
└── 20251113\
    ├── 20251113_worklog.md
    ├── blueprint_export\   # ブループリントのスクリーンショット等
    ├── automation_script\  # 自動化スクリプト
    └── log\
        └── build_log.txt
```

### Git管理の注意
以下のフォルダは Git から除外（サイズが大きいため）:
- `Binaries/`
- `Intermediate/`
- `DerivedDataCache/`
- `Saved/`

`.gitignore` 例:
```
Binaries/
Intermediate/
DerivedDataCache/
Saved/
*.sln
*.suo
```

### バージョン管理すべきもの
- `Content/`: アセット、ブループリント
- `Source/`: ソースコード
- `Config/`: 設定ファイル
- `.uproject`: プロジェクトファイル

### 作業ログに記載すべき内容
- 使用した Unreal Engine のバージョン
- 変更したブループリント・アセット
- ビルド設定の変更内容
- プラグインの追加・削除

## 新しいツールを追加する際の手順

新しいツール・プロジェクトを扱う際は、以下の手順でルールを追加してください：

1. **ツール名と概要を記載**
   - ツールの正式名称
   - バージョン情報
   - 主な用途

2. **プロジェクトファイルの管理方法を定義**
   - プロジェクトファイルの拡張子
   - 保存場所
   - バージョン管理方法

3. **フォルダ構造の例を示す**
   - 推奨されるフォルダ構成
   - 素材・アセットの配置場所
   - 出力ファイルの配置場所

4. **Git 管理の注意事項を明記**
   - Git から除外すべきフォルダ
   - バージョン管理すべきファイル
   - .gitignore の設定例

5. **特有の注意点を記載**
   - ツール固有の制約事項
   - パフォーマンス上の注意点
   - トラブルシューティング

6. **作業ログに記載すべき情報を定義**
   - バージョン情報
   - 使用したプラグイン・エクステンション
   - 設定の変更内容
   - ビルド・レンダリング設定

## 共通の注意事項

すべての特殊ツール・プロジェクトに共通する注意事項：

### ファイルサイズに関する注意
- 大容量ファイルは Git にコミットしない
- レンダリング出力、ビルド成果物は `log/` フォルダに格納
- 素材ファイルのサイズを確認し、必要に応じて外部ストレージを利用

### ドキュメント化
- プロジェクト固有の設定ファイルは文書化
- 使用したツール・プラグインのバージョン情報を記録
- 特殊な設定や手順は必ずREADMEに記載

### バージョン情報の記録
- ツールのバージョン情報を必ず記録
- プラグイン・エクステンションのバージョンも記録
- 依存関係を明確にする

### 外部依存関係の管理
- 外部ライブラリ・プラグインのリストを作成
- インストール手順をドキュメント化
- ライセンス情報を確認・記録

### 環境構築の記録
- 初回セットアップ手順をドキュメント化
- 必要なシステム要件を記載
- トラブルシューティング情報を追記

### 作業ログの記録
- 変更内容を詳細に記録
- 使用したバージョンを明記
- 問題が発生した場合は原因と解決策を記録

## トラブルシューティング

### 問題: プロジェクトファイルが開けない

**原因**:
- バージョンの不一致
- 依存ファイルの欠損
- 環境の違い

**解決策**:
1. ツールのバージョンを確認
2. 作業ログで使用バージョンを確認
3. 必要な依存ファイルを確認
4. 環境設定を確認

### 問題: 大容量ファイルでGitが遅い

**原因**:
- 大容量ファイルがGit管理下にある

**解決策**:
1. `.gitignore` を確認
2. 大容量ファイルを除外設定に追加
3. Git履歴から削除（git filter-branch等）
4. 大容量ファイルは外部ストレージに移動

### 問題: 素材ファイルが見つからない

**原因**:
- パスの違い
- 素材ファイルの移動・削除

**解決策**:
1. プロジェクトファイルで参照パスを確認
2. 素材フォルダの構造を確認
3. 作業ログで変更履歴を確認
4. プロジェクトの収集機能を使用（After Effects等）
