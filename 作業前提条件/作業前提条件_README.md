## 🎯 3つの役割と対応ドキュメント

### Worker（実装担当）

**役割**: コード実装、スクリプト作成、バグ修正などの実作業を担当
**視点**: ミクロ（個別の実装に集中）

**ドキュメント**:
- **統合版**: `作業前提条件_Worker_v2.9.md`（全内容を1ファイルで確認）
- **分割版**: `Worker_v2.9/`（必要な部分のみ読込、トークン削減）

---

### Reviewer（レビュー・評価担当）

**役割**: コードレビュー、品質チェック、作業指示書・修正指示書作成
**視点**: 品質保証（実装の妥当性を評価）

**ドキュメント**:
- **統合版**: `作業前提条件_Reviewer_v1.7.md`
- **分割版**: `Reviewer_v1.7/`

---

### Maintainer（全体管理・整備担当）

**役割**: Workspace整理、アーカイブ管理、ドキュメント体系整備、改善提案
**視点**: マクロ（全体の俯瞰・一貫性確保）

**ドキュメント**:
- **統合版**: `作業前提条件_Maintainer_v1.6.md`
- **分割版**: `Maintainer_v1.6/`

---

## 📖 統合版 vs 分割版

### 統合版の使い方

**特徴**: 全内容が1ファイルにまとまっている
**用途**:
- 初回確認時（全体像の把握）
- オフライン参照時
- 全文検索時

**ファイル**:
- `作業前提条件_Worker_v2.9.md`
- `作業前提条件_Reviewer_v1.7.md`
- `作業前提条件_Maintainer_v1.6.md`

---

### 分割版の使い方

**特徴**: モジュール単位で分割、必要な部分のみ読込可能
**用途**:
- 作業中の確認（トークン削減）
- 特定セクションのみ参照
- 頻繁な更新・メンテナンス

**フォルダ**:
- `Worker_v2.9/`
- `Reviewer_v1.7/`
- `Maintainer_v1.6/`

**推奨**: 作業時は分割版を使用（トークン効率が良い）

## 🔍 使い方の例

### 例1: Workerとして新機能を実装する

```
1. Worker_v2.9/00_QUICK_REFERENCE.md を読む（最重要ルール確認）
2. Worker_v2.9/02_実装プロセスと品質基準.md を読む（実装手順確認）
3. 作業開始
4. 必要に応じて他のモジュールを参照
```

### 例2: Reviewerとしてコードレビューを行う

```
1. Reviewer_v1.7/00_QUICK_REFERENCE.md を読む

3. Reviewer_v1.7/03_チェックリストと評価基準.md を読む
4. レビュー実施
```

### 例3: Maintainerとしてアーカイブ整理を行う

```
1. Maintainer_v1.6/00_QUICK_REFERENCE.md を読む
2. Maintainer_v1.6/02_Workspace管理.md を読む
3. アーカイブ計画・実施
```
