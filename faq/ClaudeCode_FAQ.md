# Claude Code FAQ

## Claude Code 使用時のよくある質問と回答

---

## 基本的な使い方

### Q1: Claude Codeに依頼する際の基本的な流れは？

**A:** 以下の流れで依頼するとスムーズです：

1. **DOCS/作業前提条件_Worker.md** を参照するよう指示
2. 本日の日付フォルダ（例：20251113）を確認・作成
3. 具体的な作業内容を依頼
4. 必要に応じて **DOCS/Claude_プロンプトテンプレート.md** のテンプレートを使用

**例**:
```
DOCS/作業前提条件_Worker.mdに従って、以下の作業を実施してください。
今日の日付: 20251113
作業内容: CSVファイルをJSONに変換するPythonスクリプトを作成
```

---

### Q2: ファイルを読み込んでもらう際の注意点は？

**A:**
- **絶対パス**を指定する（相対パスは避ける）
- ファイルパスは正確に記載
- スペースを含むパスは引用符で囲む

**良い例**:
```
/Users/hiraihironori/Desktop/WorkSpace/20251113/data.csv を読み込んで処理してください
```

**悪い例**:
```
data.csvを読み込んで（どのディレクトリか不明）
```

**注意**:
- macOSではパス区切りは `/`（スラッシュ）を使用します
- ホームディレクトリは `~` で表現できます（例: `~/Desktop/WorkSpace`）
- Claude Codeへの依頼時は絶対パスを使用してください

---

### Q3: エラーが発生した場合はどうすればいい？

**A:** 以下の手順で対応：

1. エラーメッセージの全文をClaude Codeに伝える
2. **DOCS/トラブルシューティング_[言語名].md** を参照するよう指示
3. エラー発生時のフォルダ（`error_YYYYMMDD_スクリプト名/`）を作成してもらう
4. 問題調査メモを作成してもらう

**例**:
```
以下のエラーが発生しました：
[エラーメッセージ]

DOCS/トラブルシューティング_Python.mdを参照して、原因を調査してください。
エラーフォルダを作成して、調査内容を記録してください。
```

## ファイル管理

### Q4: 過去の日付フォルダのファイルを修正したい

**A:** 直接編集せず、当日のフォルダに新バージョンを作成します。

**依頼例**:
```
20251112\data_process.py を修正して、20251113フォルダに新バージョンを作成してください。
細かい修正なので、ファイル名は data_process_v1.1.py としてください。
```

---

### Q5: 出力ファイルはどこに保存される？

**A:** 日付フォルダ内の **log/** サブフォルダに保存されます。

```
20251113\
└── log\
    ├── output_20251113_143052.csv
    ├── result.json
    └── error.log
```

---

### Q6: テストデータはどこに配置すればいい？

**A:** 日付フォルダ内の **test_data/** サブフォルダに配置します。

```
20251113\
└── test_data\
    ├── sample_input.csv
    ├── expected_output.csv
    └── test_config.json
```

---

## 作業ログ

### Q7: 作業ログは必ず作成する必要がある？

**A:** はい、**必須**です。各日付フォルダには `YYYYMMDD_worklog.md` を作成してください。

**依頼例**:
```
本日の作業ログ（20251113_worklog.md）を作成してください。
今日の作業内容をまとめてください。
```

---

### Q8: 作業ログには何を書けばいい？

**A:** 以下の項目を含めます：

- 作業目的
- 実施内容
- 作成・修正したファイル
- 課題・問題点
- 次回タスク
- メモ・備考

テンプレートは **DOCS/作業前提条件_Worker.md** に記載されています。

---

## バージョン管理

### Q9: バージョン番号のルールは？

**A:**
- **マイナーバージョン（v1.1, v1.2）**: 細かい修正・バグ修正
- **メジャーバージョン（v2, v3）**: 大きな変更・機能追加

**例**:
```
元ファイル: data_process.py
細かい修正: data_process_v1.1.py
大きな変更: data_process_v2.py
```

---

### Q10: どのタイミングでバージョンを上げるべき？

**A:**

**マイナーバージョン**:
- バグ修正
- 軽微な追記・修正
- コメントの追加

**メジャーバージョン**:
- 機能の大幅な追加・削除
- アルゴリズムの根本的な変更
- 互換性を損なう変更

## 複雑な依頼

### Q11: 複数のタスクを同時に依頼できる？

**A:** 可能ですが、明確に分けて依頼してください。

**依頼例**:
```
以下の3つのタスクを実施してください：
1. CSVファイルをJSONに変換するスクリプト作成
2. 既存のPythonスクリプトのリファクタリング
3. 作業ログの作成
```

---

### Q12: 段階的に依頼したほうがいい？

**A:** 複雑な作業の場合は段階的に依頼することを推奨します。

**例**:
```
# 第1段階
まず、データを読み込んで表示するスクリプトを作成してください。

# 第2段階（第1段階の確認後）
次に、データを加工する処理を追加してください。

# 第3段階
最後に、JSONで出力する機能を追加してください。
```

---

## テンプレート活用

### Q13: プロンプトテンプレートはどう使う？

**A:** **DOCS/Claude_プロンプトテンプレート.md** から適切なテンプレートをコピーして、具体的な内容に置き換えます。

**例**:
```
DOCS/Claude_プロンプトテンプレート.mdの「データ処理スクリプト作成」テンプレートを使って依頼します。

Pythonでcsvファイルを読み込んで、特定の列でフィルタリングを行い、JSONで出力するスクリプトを作成してください。
...（以下、テンプレートに従って記入）
```

---

### Q14: コードテンプレートを活用できる？

**A:** はい、**batch_file/templates/** に基本テンプレートがあります。

**依頼例**:
```
batch_file/templates/template_python.py をベースに、
CSVファイルを処理するスクリプトを作成してください。
```

## ツール固有の質問

### Q15: After Effectsの自動化スクリプトを依頼できる？

**A:** 可能です。以下の情報を含めて依頼してください：

- 自動化したい作業内容
- 対象プロジェクトのパス
- 出力設定（形式、解像度など）

**依頼例**:
```
After Effectsで以下の作業を自動化するスクリプトを作成してください。
作業内容: レンダーキューに複数のコンポジションを追加
対象プロジェクト: 20251113\project.aep
出力形式: MP4, 1920x1080, 30fps
```

参考: **DOCS/トラブルシューティング_AfterEffects.md**

---

### Q16: Unreal Engineのブループリントを自動化できる？

**A:** Python Editorスクリプトによる自動化が可能です。

**依頼例**:
```
Unreal Engineで以下の作業を自動化するPythonスクリプトを作成してください。
作業内容: 特定のアセットを一括でインポート
使用バージョン: Unreal Engine 5.3
```

参考: **DOCS/トラブルシューティング_UnrealEngine.md**

## ファイルの整理

### Q17: 古いファイルを削除したい

**A:** 直接削除せず、**reject_file/** フォルダに移動してください。

**依頼例**:
```
20251101フォルダを reject_file/ に移動してください。
30日以上経過しているため、削除候補です。
```

---

### Q18: reject_file フォルダのファイルはいつ削除する？

**A:** 手動で定期的（月1回程度）に確認して削除します。Claude Codeには依頼しません。

## コーディング規約

### Q21: どのコーディング規約に従うべき？

**A:** 言語ごとに以下のドキュメントがあります：

- **DOCS/コーディング規約_Python.md**（PEP 8準拠）
- **DOCS/コーディング規約_PowerShell.md**
- **DOCS/コーディング規約_Batch.md**

**依頼例**:
```
DOCS/コーディング規約_Python.mdに従って、
以下のコードをリファクタリングしてください。
```

---

### Q22: コーディング規約に違反しているか確認できる？

**A:** はい、レビューを依頼できます。

**依頼例**:
```
20251113\script.py が DOCS/コーディング規約_Python.md に
準拠しているか確認してください。
問題があれば修正案を提示してください。
```

## その他

### Q23: Claude Codeがうまく理解してくれない

**A:** 以下を試してください：

1. **具体的に指示**する（ファイルパス、処理内容を明確に）
2. **段階的に依頼**する
3. **DOCS/作業前提条件_Worker.md** を参照するよう明示
4. **例を示す**（期待する結果のサンプル）

---

### Q24: 同じ依頼を繰り返し行う場合は？

**A:** スクリプト化または **DOCS/Claude_プロンプトテンプレート.md** に独自テンプレートを追加します。

**依頼例**:
```
DOCS/Claude_プロンプトテンプレート.mdに、
「毎週の集計レポート作成」テンプレートを追加してください。
```

---

### Q25: 設定ファイルに機密情報を含めたくない

**A:** **DOCS/作業前提条件_Worker.md** の「環境変数・設定ファイルの管理」セクションを参照してください。

- `.env` ファイルを使用
- `.gitignore` に追加
- `.env.example` をテンプレートとして作成

---

### Q26: どのドキュメントを優先的に読むべき？

**A:** 以下の優先順位で：

1. **DOCS/作業前提条件_Worker.md**（実装・作業担当用、最重要）
2. **DOCS/Claude_プロンプトテンプレート.md**
3. 言語別の**クイックリファレンス**
4. 言語別の**トラブルシューティング**
5. **コーディング規約**
6. **DOCS/依存関係図.md**（ツール・バージョン確認時）
