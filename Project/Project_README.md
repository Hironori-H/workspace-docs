## 📝 ファイル管理ルール

### ファイル名の規則

```
{内容}_{バージョン}.md

例:
- 用語集.md
- 依存関係図.md
- 役割別業務早見表_v2.md
```

### バージョン管理

- **用語集**: バージョン番号なし（随時更新）
- **業務早見表**: バージョン番号あり（v1, v2, v3...）
- **ガイド系**: 日付で管理（最終更新日を記載）

### 更新時のルール

1. ファイル内の「最終更新日」を必ず更新
2. 大幅な変更の場合は、changelogフォルダに変更履歴を記録
3. 他のドキュメントへの影響を確認（特に作業前提条件での参照）
