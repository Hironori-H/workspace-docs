# 役割別業務早見表 v2.0

**作成日**: 2025-11-23
**目的**: ユーザーが「どの役割・どのモデルに依頼するか」を判断するための実用的なガイド

## 基本原則

| 役割 | 責務 | 視点 | モデル候補 |
|------|------|------|------------|
| **Worker** | 実装・作業を担当（手を動かす） | ミクロ | Sonnet/Haiku |
| **Reviewer** | レビュー・評価を担当（チェックする） | 品質保証 | Haiku/Sonnet |
| **Maintainer** | 全体管理・整備を担当（俯瞰する） | マクロ | Opus |

## 📌 使い方

### ステップ1: 作業内容を特定
「何をしたいか」を明確にする

### ステップ2: 役割を選択
上記の表から適切な役割を選ぶ

### ステップ3: モデルを選択
- **簡単・定型的** → Haiku (コスト0.2x)
- **標準的・創造的** → Sonnet (コスト1x)
- **重要・戦略的** → Opus (コスト5x)

### ステップ4: 依頼実行
```bash
# 例：新機能実装をWorkerに依頼
claude --model sonnet

# 例：コードレビューをReviewerに依頼
claude --model haiku

# 例：技術選定をMaintainerに依頼
claude --model opus
```

## ⚠️ 注意事項

1. **役割の境界を守る**
   - Reviewerは実装しない（修正指示書を作成）
   - Maintainerは個別実装しない（計画を立てWorkerに依頼）
   - Workerはレビュー権限なし（Reviewerに依頼）

2. **コストと品質のバランス**
   - 安易にHaikuを選ばない
   - 重要な判断にはOpusを使う
   - 役割定義を優先する

3. **フロー例：バグ修正**
   ```
   1. Reviewer(Haiku/Sonnet): バグ調査・原因特定
   2. Reviewer(Haiku): 修正指示書作成
   3. Worker(Haiku/Sonnet): 修正実装
   4. Reviewer(Haiku): 修正確認
   ```
