# FAQ - よくある質問

**最終更新日**: 2025年11月17日
**バージョン**: 1.0

---

## 全般

### Q1: このプロジェクトの目的は何ですか？

**A**: Adobe After Effects と Unreal Engine の作業を自動化し、デザイナーの作業効率を大幅に向上させることが目的です。具体的には以下を実現します：
- .aepファイルの自動処理
- レイヤー情報のデータベース化
- レンダリングの自動化
- 作業プロセスの標準化

---

### Q2: Claude Codeの役割は何ですか？

**A**: Claude Codeは3つの役割を担当します：
1. **Worker**: 機能の実装、スクリプトの作成
2. **Reviewer**: 成果物のレビュー、品質評価
3. **Maintainer**: Workspace全体の管理、ドキュメント整備

各役割の詳細は `DOCS/作業前提条件_[役割名].md` を参照してください。

---

### Q3: 新しいタスクを開始する際の手順は？

**A**: 以下の手順で開始してください：
1. 本日の日付フォルダを作成（例: `20251117`）
2. 作業ログ（`YYYYMMDD_worklog.md`）を作成
3. 作業目的、実施内容を記録
4. 実装・作業を実施
5. 作業ログを更新

詳細は `作業前提条件_Worker.md` を参照してください。

---

## Workspace管理

### Q4: 日付フォルダはいつ作成しますか？

**A**: 作業を開始する日に作成します。作業が発生しない日はフォルダを作成する必要はありません。

**フォルダ名形式**: `YYYYMMDD`（例: 20251117）

---

### Q5: 古い日付フォルダはどうすればよいですか？

**A**: 以下のタイミングでアーカイブします：
- 作業が完了してから30日以上経過した場合
- 月末に前月のフォルダをまとめてアーカイブ

**アーカイブ手順**:
```powershell
# アーカイブスクリプトを実行
.\batch_file\archive_november.ps1
```

アーカイブ後は `archive/YYYY/MM-MonthName/YYYYMMDD/` に移動されます。

---

### Q6: reject_file フォルダの用途は？

**A**: 却下された成果物（修正指示書、スクリプト等）を保管します。学習・参考資料として活用できます。

**配置場所**: 各日付フォルダ内（例: `20251117/reject_file/`）

---

### Q7: batch_file フォルダの用途は？

**A**: プロジェクト全体で共通で使用するスクリプト・バッチファイルを保管します。

**主な用途**:
- Unreal Engine の起動スクリプト
- システム修正スクリプト
- アーカイブスクリプト
- テンプレートファイル（`templates/` 内）

## After Effects 自動化

### Q8: After Effects の自動化はどのように動作しますか？

**A**: 以下の流れで動作します：
1. Node.js の automation システムが .aep ファイルを監視
2. ファイル検出後、After Effects を起動
3. SimpleTrigger.jsx がレイヤー情報を抽出
4. JSON形式で出力
5. PostgreSQL データベースに格納

詳細は `20251117/AE_dialog_suppression_implementation_summary.md` を参照してください。

---

### Q9: ダイアログが表示されてしまいます。どうすればよいですか？

**A**: Phase 1 のダイアログ抑制機能が実装済みです。以下を確認してください：
1. SimpleTrigger.jsx が最新版か確認
2. aeExecutor.js に `-noerr` 引数が含まれているか確認
3. 環境設定ファイルでフォント警告抑制が有効か確認

それでも解決しない場合は、作業ログに詳細を記録し、ユーザー様に報告してください。

---

### Q10: JSON出力に問題があります。どう対処すればよいですか？

**A**: 以下の確認を行ってください：
1. SimpleTrigger.jsx のログファイルを確認
   - 場所: `/Users/hiraihironori/Library/Application Support/AE_Monitor/Logs/SimpleTrigger_YYYYMMDD.log`
2. エラーメッセージを確認
3. 作業ログに問題を記録
4. Reviewer にレビューを依頼

現在、いくつかの既知の問題があります（例: boxTextSize エラー）。詳細は作業ログを参照してください。

詳細なトラブルシューティングは作業ログを参照してください。

## 作業プロセス

### Q13: レビュープロセスはどのように進めますか？

**A**: 以下の流れで進めます：
1. Worker が実装・作業を完了
2. Worker が作業ログを更新
3. Reviewer がレビューを実施
4. Reviewer がレビュー結果を記録（`レビュー結果_YYYYMMDD_[対象名].md`）
5. 問題がある場合、Reviewer が修正指示書を作成
6. Worker が修正指示書に従って修正
7. 再度レビュー → 承認

詳細は `作業前提条件_Reviewer.md` を参照してください。

---

### Q14: 修正指示書に従って修正しましたが、まだ問題があります。

**A**: 以下の対応を行ってください：
1. 問題の内容を作業ログに詳細に記録
2. エラーメッセージ、ログファイルを添付
3. Reviewer に再度レビューを依頼
4. 必要に応じて、ユーザー様に判断を仰ぐ

（注: 複数回の試行が必要な場合もあります。例: JSON出力改善 v1〜v5）

---

### Q15: 作業ログには何を記録すればよいですか？

**A**: 以下の内容を記録してください：
- 作業目的
- 実施内容
- 作成・修正したファイル
- 課題・問題点
- テスト結果（実施した場合）
- 次回タスク

テンプレートは `batch_file/templates/` を参照してください（今後追加予定）。

## 技術的な質問

### Q16: ExtendScript（.jsx）のデバッグ方法は？

**A**: 以下の方法でデバッグできます：
1. `$.writeln()` でコンソール出力
2. SimpleTrigger.jsx の `writeLog()` 関数でログファイルに記録
3. After Effects の ExtendScript Toolkit を使用（ただし、古いバージョンのみ）
4. ログファイルを確認
   - `C:\Users\CLPC-63\AppData\Roaming\AE_Monitor\Logs\SimpleTrigger_YYYYMMDD.log`

---

### Q17: Python スクリプトの文字エンコーディングは？

**A**: 以下の規則に従ってください：
- **Pythonスクリプト**: UTF-8
- **バッチファイル**: UTF-8 with BOM または Shift-JIS
- **PowerShell**: UTF-8 with BOM

詳細は `作業前提条件_Worker.md` のコーディング規約セクションを参照してください。

## トラブルシューティング

### Q19: Git で削除したファイルが git status に残っています。

**A**: 以下のコマンドで解決します：
```bash
git add .
git commit -m "Remove deleted files"
```

または、特定のファイルのみ削除を記録する場合：
```bash
git rm [ファイル名]
git commit -m "Remove [ファイル名]"
```
