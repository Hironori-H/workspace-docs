# 📚 Reference

よく参照する技術リファレンス・クイックリファレンスを置く場所です。

---

## 📌 このフォルダの目的

ユーザーがコーディングやGit管理などの作業をする際に、**すぐに参照できる技術資料**を集約しています。

**対象者**: ユーザー（開発者）
**使用場面**: コーディング中、Git操作中、コマンド確認時

---

## 📁 置くべきもの

| 種類 | 説明 | 例 |
|:---|:---|:---|
| **クイックリファレンス** | よく使うコマンド・構文の早見表 | `クイックリファレンス_Git.md`、`クイックリファレンス_Python.md` |
| **コーディング規約** | 言語別のスタイルガイド・ベストプラクティス | `コーディング規約_Python.md`、`コーディング規約_ShellScript.md` |
| **コマンドリファレンス** | プロジェクト固有のコマンド集 | `コマンドリファレンス_ログ保存.md` |
| **業務フロー** | Reviewer/Worker/Maintainer用の業務早見表 | `Reviewer_クイックリファレンス.md` |

---

## ❌ 置かないもの

| 種類 | 理由 | 適切な保存先 |
|:---|:---|:---|
| プロジェクト固有の用語集 | プロジェクト管理情報のため | `DOCS/Project/` |
| 依存関係図・システム構成図 | プロジェクト管理情報のため | `DOCS/Project/` |
| 作業前提条件ドキュメント | Claude専用のため | `DOCS/作業前提条件/` |
| 設定手順・トラブルシューティング | 個人のナレッジベースのため | `Obsidian/My_Wisdom/40_Reference/` |

## 🔍 使い方

### ユーザー（開発者）の場合

1. **コーディング中**: コーディング規約を参照
2. **Git操作中**: クイックリファレンスを参照
3. **ログ保存時**: コマンドリファレンスを参照

### Claude（Worker/Reviewer）の場合

1. **業務フロー確認**: Reviewer_クイックリファレンス.mdを参照
2. **コマンド実行**: コマンドリファレンスを参照
